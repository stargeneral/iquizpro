## Naming & Code Conventions (Mandatory)

These apply to every file created or modified in this phase:

**Globals / modules:**
- Window globals: `window.QuizPros[ModuleName]` (PascalCase) — e.g. `window.QuizProsPremium`
- File names: `kebab-case.js` — e.g. `premium-guard.js`
- CSS classes and IDs: `kebab-case` — e.g. `.premium-badge`, `#upgrade-modal`
- Constants: `UPPER_SNAKE_CASE`
- Functions and variables: `camelCase`
- JSON template files: `kebab-case.json`

**Module pattern (mandatory for all new JS modules):**
```javascript
window.QuizPros[Name] = (function() {
  // Private state
  const config = window.QuizProsConfig;
  const utils  = window.QuizProsUtils;

  // Private functions
  function privateHelper() { ... }

  // Public API
  return {
    initialize: initialize,
    publicMethod: publicMethod
  };
})();
```

**File locations:**
- Business logic modules: `js/modules/`
- UI components: `js/components/`
- Utilities: `js/utils/`
- Stylesheets: `css/`
- Quiz data / templates: `templates/`
- Dev docs: `docs/development/`
- Images / fonts / sounds: `assets/`
- **Cloud Functions**: `functions/` (new in Phase 3 — Node.js, server-side only)

**Dependency rules:**
- Never create circular dependencies
- All modules may depend on: `QuizProsConfig`, `QuizProsUtils`, `QuizProsStorage`
- Components may depend on modules; **modules must NEVER depend on components**
- New frontend modules must be added to `src/app-entry.js` (not `index.html`) and listed in CLAUDE.md's dependency graph
- Cloud Functions (`functions/index.js`) are server-side Node.js — they do NOT use the IIFE pattern

**Change safety:**
- Never rename a `window.QuizPros*` global without searching the entire codebase first
- Never remove a public API method without verifying zero callers
- When splitting a file, keep the original global as a facade
- Every new file must have a JSDoc header comment explaining its purpose

### 3.1 — Cloud Functions Infrastructure (Prerequisite for 3.2 and 3.3)

Both AI quiz generation (3.2) and Stripe webhooks (3.3) require Firebase Cloud Functions. Set up the infrastructure once.

**Initialise Cloud Functions:**
```bash
# In project root:
firebase init functions
# Select: JavaScript (not TypeScript)
# ESLint: No
# Install dependencies: Yes
```

This creates:
```
functions/
  index.js        ← all Cloud Function exports go here
  package.json    ← server-side dependencies (openai/gemini, stripe, etc.)
```

**Update `firebase.json`** to add the functions source:
```json
{
  "functions": { "source": "functions" },
  "hosting": { ...existing config... }
}
```

**Deploy commands (Phase 3+):**
```bash
firebase deploy --only functions          # functions only
firebase deploy --only hosting            # hosting only
firebase deploy                           # both together
npm run deploy                            # build + deploy hosting (existing script)
```

> ⚠️ **Billing**: Cloud Functions require Firebase **Blaze (pay-as-you-go)** plan. Verify the `quizpros` project is on Blaze before deploying functions (`console.firebase.google.com` → project settings → Usage and billing).

> ⚠️ **API keys**: Store secret keys (AI API key, Stripe secret key) in Firebase Functions environment config or Google Secret Manager — **never in frontend code**.
> ```bash
> firebase functions:config:set openai.key="sk-..." stripe.secret="sk_test_..."
> # Access in functions: functions.config().openai.key
> ```

### 3.2 — AI Quiz Generation (Complete Scaffolded Feature)

The "Generate Quiz with AI" button is wired in `live-presenter.html` and calls Firebase Cloud Functions. It has **never worked** — two issues must be fixed:

**Frontend fix** — add missing Firebase Functions SDK to `live-presenter.html` (one line, before the existing Firebase scripts close):
```html
<script src="https://www.gstatic.com/firebasejs/9.20.0/firebase-functions-compat.js"></script>
```
Without this, `firebase.functions()` is undefined and `AIQuizService` throws `"Firebase Functions service is required"` before the modal can open.

**Backend** — implement these callable functions in `functions/index.js`:

| Function name | What it does |
|--------------|-------------|
| `generateQuiz` | Authenticates user, checks premium, calls AI API, saves quiz to Firestore, returns quiz JSON |
| `getUserUsageStats` | Returns `{ currentLimit: { used, limit, remaining, tier } }` for rate-limit display |
| `getGeneratedQuiz` | Fetches a previously generated quiz by ID from Firestore |
| `listUserQuizzes` | Lists all quizzes generated by the authenticated user |
| `deleteGeneratedQuiz` | Deletes a quiz document from Firestore |
| `generateAdaptiveQuestion` | Generates a single follow-up question during a live session |

**Expected response shape** (from reading the live-presenter bundle):
```javascript
// generateQuiz — success response
{ success: true, quiz: { id, title, questions: [...] }, usage: { used, limit, tier } }

// getUserUsageStats — success response
{ success: true, currentLimit: { used, limit, remaining, tier } }
```

**Rate limiting tiers** (already wired in frontend):
- Free tier: limited quiz generations per month (define the limit)
- Premium tier: higher or unlimited

**AI provider decision needed from user**: The frontend doesn't specify which AI API to use — that's a server-side implementation detail. Options:
- **Google Gemini** — free tier available, same Google ecosystem as Firebase, `@google/generative-ai` npm package
- **OpenAI GPT-4o-mini** — widely documented, `openai` npm package, cost-effective

The user must provide their chosen API key before this can be implemented.

// Stripe webhook — handles subscription lifecycle events
exports.stripeWebhook = functions.https.onRequest(async (req, res) => { ... });

//   checkout.session.completed  → mark user as premium in Firestore

**Firestore write after payment:**
```
users/{uid}/subscription: {
  tier: "basic" | "plus",
  stripeCustomerId: "cus_...",

  expiresAt: Timestamp,
  updatedAt: Timestamp
}
```

## Known Issues & Constraints

1. **Cloud Functions require Blaze billing** — Firebase free (Spark) plan does not support Cloud Functions deployment. Verify billing before starting 3.1.

2. **`live-presenter.html` is a static file, not a webpack template** — It is copied to `dist/` by CopyPlugin unchanged. Edit the root `live-presenter.html`; run `npm run build` to propagate changes to `dist/`.

3. **Live presenter JS bundle is pre-built** — `js/live-presenter.e6fc5f16770ec082babb.js` is minified and cannot be modified directly. The AI quiz generation feature's frontend logic (modal, form, UI) is already baked into this bundle. Only the missing SDK line in `live-presenter.html` and the backend Cloud Functions need to be added.

4. **Firestore security rules** — Verify existing rules at Firebase Console before writing new paths (`users/{uid}/subscription`, `users/{uid}/quizHistory`, `generatedQuizzes/{quizId}`). Add rules for new collections as part of Phase 3 work.

5. **Old auth scripts on disk** — Not loaded, safe to delete. Do this in Step 0 to keep the codebase clean.

6. **`src/index.template.html` vs `index.html`** — `src/index.template.html` is the source; `dist/index.html` is generated. Always edit `src/index.template.html` for main app HTML changes.

7. **`"Your Score: /"` display** — Personality quizzes show a blank score (no numeric score by design). Pre-existing, low priority, not a Phase 3 concern.

## How to Work

1. **Read the required files first** (CLAUDE.md, handoff docs, DEVELOPMENT_PLAN.md)
2. **Use Desktop Commander** to read and edit files on the local machine
3. **Make targeted, incremental changes** — one sub-task at a time
4. **Build before testing**: `npm run build` → test in browser (or `npm start` for dev server with hot reload)
5. **Source files are in `js/`, `css/`, `src/`** — do NOT edit files inside `dist/` directly; they are overwritten on every build
6. **Module loading order** is defined in `src/app-entry.js` (not `index.html`) — preserve import order when adding new modules
7. **Search the entire codebase** before renaming or removing any `window.QuizPros*` global
8. **After each sub-task**, confirm with user before moving to the next
9. **Deploy to preview channel first**: `firebase hosting:channel:deploy phase3-test`
10. **Then production**: `npm run deploy` (or `npm run build && firebase deploy`)
